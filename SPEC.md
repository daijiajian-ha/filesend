# FileSend 企业版 - 设计规范文档

> 项目名称：FileSend（简称 FileSend）
> 版本：v1.0
> 日期：2026-04-16
> 状态：初稿

---

## 1. 项目概述

### 1.1 背景与目标

**使用场景**：企业内换机时，两台PC（Windows/macOS）之间的数据传输

**核心目标**：在 FileSend 基础上，简化功能、强化安全、做企业专用版本

### 1.2 平台支持

| 平台 | 支持 | 说明 |
|------|------|------|
| Windows 10+ | ✅ | 主要平台 |
| macOS 12+ | ✅ | 主要平台 |
| iOS/Android | ❌ | 不支持 |

---

## 2. 功能范围

### 2.1 保留功能（来自 FileSend）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 文件传输 | 单文件/文件夹传输 | P0 |
| 文件夹传输 | 直接传整个文件夹 | P0 |
| 跨平台 | Windows ↔ macOS | P0 |
| HTTPS 加密 | 端到端加密 | P0 |

### 2.2 新增功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 数字口令验证 | 6位数字，一次性使用 | P0 |
| 断点续传 | 大文件传输中断后继续 | P0 |
| 传输日志 | 详细记录，含设备信息 | P0 |
| 完成弹窗通知 | 传输完成后弹窗提醒 | P1 |

### 2.3 删除功能（FileSend 原生但不需要）

| 功能 | 原因 |
|------|------|
| 消息传输 | 换机场景不需要 |
| 多语言 | 只做中文 |
| 多设备自动发现列表 | 简化为扫码/输入IP直连 |
| 复杂设置页 | 企业定制无需配置 |
| 多平台App (iOS/Android) | 仅PC端 |

## 4. 数据流程

### 4.1 传输流程（发送方视角）

```
1. [用户] 选择文件/文件夹
         │
2. [系统] 生成6位数字口令
         │
3. [系统] 打包成 .zip（文件夹才打包，单独文件直接传）
         │
4. [系统] 计算文件hash（SHA256）
         │
5. [UI] 显示口令 + 文件信息 + 二维码（可选）
         │
6. [用户] 在接收方输入口令
         │
7. [系统] 接收方验证通过，开始传输
         │
8. [传输层] 分块传输（每块1MB），记录进度
         │
9. [完成] 弹窗通知 + 写日志
```

### 4.2 接收流程（接收方视角）

```
1. [用户] 输入口令 / 扫码
         │
2. [系统] 发送验证请求到发送方
         │
3. [发送方] 验证口令，返回文件信息
         │
4. [用户] 选择保存位置
         │
5. [系统] 开始接收文件块
         │
6. [系统] 校验hash
         │
7. [系统] 解压（如是zip）
         │
8. [完成] 弹窗通知 + 写日志
```

### 4.3 断点续传流程

```
传输前：
  - 发送方保存：文件hash + 已传字节数
  - 接收方保存：已收字节数

中断后：
  - 重连 -> 验证口令 -> 发送 "offset: XXXXX" 
  - 发送方从 offset 继续传
  - 接收方从 offset 继续收
```

## 5. 接口设计

### 5.1 核心接口

#### 5.1.1 发起传输请求
```
POST /api/transfer/init
Body: {
  "code": "123456",           // 6位口令
  "fileName": "doc.zip",
  "fileSize": 1048576,
  "fileHash": "sha256:xxx",
  "senderInfo": {
    "ip": "192.168.1.100",
    "mac": "AA:BB:CC:DD:EE:FF",
    "hostname": "DESKTOP-ABC"
  }
}
Response: {
  "transferId": "uuid",
  "accepted": true
}
```

#### 5.1.2 获取传输信息
```
GET /api/transfer/{transferId}/info
Response: {
  "fileName": "doc.zip",
  "fileSize": 1048576,
  "senderInfo": {...}
}
```

#### 5.1.3 传输文件块
```
POST /api/transfer/{transferId}/chunk
Headers: {
  "Content-Range": "bytes {start}-{end}/{total}"
}
Body: <binary data>
Response: {
  "received": bytes
}
```

#### 5.1.4 完成传输
```
POST /api/transfer/{transferId}/complete
Response: {
  "success": true,
  "hashVerified": true
}
```

## 6. 日志设计

### 6.1 日志格式

```json
{
  "timestamp": "2026-04-16T10:43:21.123+08:00",
  "event": "transfer_start",
  "transferId": "uuid-xxx",
  "direction": "send",
  "sender": {
    "ip": "192.168.1.100",
    "mac": "AA:BB:CC:DD:EE:FF",
    "hostname": "DESKTOP-ABC"
  },
  "receiver": {
    "ip": "192.168.1.101",
    "mac": "11:22:33:44:55:66",
    "hostname": "MACBOOK-PRO"
  },
  "file": {
    "name": "document.zip",
    "size": 1245184000,
    "hash": "sha256:xxx"
  }
}
```

### 6.2 事件类型

| 事件 | 说明 |
|------|------|
| transfer_start | 开始传输 |
| transfer_progress | 进度更新（每5%记一条） |
| transfer_pause | 暂停 |
| transfer_resume | 恢复 |
| transfer_complete | 完成 |
| transfer_error | 错误 |
| code_verified | 口令验证通过 |
| code_invalid | 口令错误 |

### 6.3 日志存储

| 项目 | 说明 |
|------|------|
| 路径 | `~/.FileSend/logs/` |
| 格式 | JSON Lines（按日期分文件） |
| 文件名 | `FileSend-2026-04-16.log` |
| 保留 | 30天自动清理 |

## 8. 开发工作量评估

### 8.1 模块拆分

| # | 模块 | 功能 | 复杂度 |
|---|------|------|--------|
| 1 | UI框架 | 发送/接收双页面 | 低 |
| 2 | 文件选择 | 原生文件选择器 | 低 |
| 3 | 打包服务 | zip压缩/解压 | 低 |
| 4 | 口令系统 | 生成+验证 | 低 |
| 5 | 传输服务 | 分块+HTTPS | 中 |
| 6 | 断点续传 | offset记录+恢复 | 中 |
| 7 | 日志系统 | JSON日志+文件 | 低 |
| 8 | 通知系统 | 弹窗通知 | 低 |
| 9 | 设备信息 | IP/MAC/主机名 | 低 |

### 8.2 优先级排序

```
第一阶段（核心功能）：
  1. UI框架
  2. 文件选择
  3. 打包服务
  4. 口令系统
  5. 传输服务（不含断点）

第二阶段（完善功能）：
  6. 断点续传
  7. 日志系统

第三阶段（体验优化）：
  8. 通知系统
  9. 设备信息展示
```

### 8.3 开发周期估算

| 阶段 | 工作量 | 说明 |
|------|--------|------|
| 第一阶段 | 3-4天 | 两人天/工作量 |
| 第二阶段 | 2-3天 | 断点续传较复杂 |
| 第三阶段 | 1-2天 | 收尾 |
| **总计** | **6-9人天** | |
